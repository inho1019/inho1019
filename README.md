<summary>튜플 타입을 사용해야하는 이유</summary>
<br/>
<h2 data-ke-size="size26">튜플 타입이란?</h2>
<p data-ke-size="size16">순서와 각 인덱스의 타입이 고정된 배열타입 ex:) array: [string, number]</p>
<h2 data-ke-size="size26">의문점</h2>
<p data-ke-size="size16">저렇게 쓸거면 그냥 객체로 사용하는게 좋지 않나? 왜 굳이 튜플로 사용하는건가? ex:) { name: string, age: number }</p>
<h2 data-ke-size="size26">답</h2>
<p data-ke-size="size16">튜플의 특징은 사용자가 이름을 지정할수 있다는 거 다양한 상황에서 활용되는 변수, 함수에 상황에 따라 임의의 이름을 설정하여 사용이 가능하다.</p>
<h3 data-ke-size="size23">우리가 자주 볼수 있는 예시로는...</h3>
<ul style="list-style-type: disc;" data-ke-list-type="disc">
<li>const [name, set] = useState()</li>
<li>const [name, {&hellip;}] = useMutation()</li>
</ul>
<p data-ke-size="size16">등등&hellip;</p>
<p data-ke-size="size16">&nbsp;</p>
<p data-ke-size="size16">이것은 API 설계원칙과 동일하다.<br />상황에 맞춰 잘활용하면 좀 더 좋은 코드를 짜는데 도움이 될것 같다</p>
</details>
